Contact Form Troubleshooting Guide

Issue: PHP Code Displayed Instead of Form Processing

When submitting the contact form, you see raw PHP code instead of the form processing.

Root Cause

This issue occurs when your web server is not configured to execute PHP files. Instead of running the PHP code, the server sends the PHP code as plain text to your browser.

Solutions

1. If Using Local Development Server

Problem: Opening HTML files directly in browser using file:// protocol won't process PHP.

Solution: Use a proper local development server:

  • XAMPP (Windows/Mac/Linux) - Download here
  • WAMP (Windows) - Download here
  • MAMP (Mac/Windows) - Download here

Place your website files in the server's document root (e.g., htdocs for XAMPP) and access via http://localhost/

2. If Using Web Hosting

Problem: Hosting account not configured for PHP or files in wrong location.

Solutions:

  • Contact your hosting provider to confirm PHP is enabled on your account
  • Check if your hosting requires specific file extensions or directories for PHP
  • Ensure files are uploaded to the correct directory (usually public_html or www)

3. Test PHP Configuration

Run these tests to diagnose the issue:

  1. Visit test_php.php - Should display "PHP is working correctly!"
  2. Visit test_mail.php - Should show if mail function works

If these don't work, it confirms the server configuration issue.

4. Alternative Solution: Use Formspree

If you can't fix the server configuration, you can use a third-party service like Formspree:

  1. Sign up at formspree.io
  2. Get your form endpoint URL
  3. Replace the form action in contact.html with your Formspree endpoint

Need Help?

If you're still having issues after trying these solutions, please contact your hosting provider or system administrator for assistance with PHP configuration.
